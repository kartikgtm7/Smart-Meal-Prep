[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
(pause)
Let me walk you through how this works under the hood. 
When you send a message, it first enters our Security Checkpoint node. This checkpoint scrubs out any PII like phone numbers, checks for malicious prompt injections, and makes sure the query is food-related.
If safe, it routes the sanitized message to our Kitchen Orchestrator agent.
The Orchestrator agent then delegates work using two specialized sub-agents: the Meal Planner agent and the Grocery List Generator agent.
The Meal Planner uses MCP server tools to check your pantry and search existing recipe files.
The Grocery List Generator extracts needed ingredients and runs an MCP tool to save them straight to a local file.
Finally, the workflow pauses at the Approve Meal Plan node, waiting for a human-in-the-loop decision before finishing.

Now, the workflow is pausing here at this human-in-the-loop checkpoint, asking: "Do you approve?"
